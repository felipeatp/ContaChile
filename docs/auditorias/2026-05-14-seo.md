# Auditoria SEO - ContaChile

**Fecha:** 2026-05-14
**Rama:** feat/dte-engine
**Scope:** apps/web

---

## Resumen

La base SEO esta bien implementada (meta tags globales, Open Graph, sitemap, robots, JSON-LD), pero hay gaps en paginas privadas, accesibilidad y datos estructurados.

---

## Hallazgos

### Críticos (P0)

#### 1. Paginas privadas sin `robots: { index: false }`
**Archivos:** `dashboard/page.tsx`, `documents/page.tsx`, `emit/page.tsx`, `settings/page.tsx`, `documents/[id]/page.tsx`
**Problema:** Ninguna pagina privada exporta metadata con `robots: { index: false }`. El `title.template` del layout genera titulos genericos.
**Fix:**
```tsx
import type { Metadata } from "next"

export const metadata: Metadata = {
  title: "Dashboard",
  description: "Resumen de tu actividad de facturacion electronica en ContaChile.",
  robots: { index: false, follow: false },
}
```

#### 2. `robots.ts` tiene `/home` en vez de `/`
**Archivo:** `apps/web/app/robots.ts`
**Problema:** La ruta `/home` no existe; la landing esta en `/`.
**Fix:**
```tsx
allow: ['/', '/login', '/sign-up'],
```

#### 3. Labels sin `htmlFor` + inputs sin `id`
**Archivos:** `settings/page.tsx`, `emit-form.tsx`
**Problema:** Todos los `<label>` usan texto plano sin atributo `htmlFor`. Esto rompe accesibilidad (screen readers no pueden asociar label con input) y es una violacion de WCAG 2.1 AA.
**Fix:**
```tsx
// Antes
<div>
  <label className="text-sm font-medium">RUT</label>
  <Input placeholder="76.123.456-7" />
</div>

// Despues
<div>
  <label htmlFor="rut" className="text-sm font-medium">RUT</label>
  <Input id="rut" placeholder="76.123.456-7" />
</div>
```

### Importantes (P1)

#### 4. JSON-LD tiene `aggregateRating` ficticio
**Archivo:** `apps/web/app/page.tsx`
**Problema:** `ratingValue: "4.8"` y `ratingCount: "127"` son datos ficticios. Google penaliza structured data con ratings falsos.
**Fix:** Eliminar `aggregateRating` hasta tener datos reales de usuarios verificados.

#### 5. Sitemap incluye `/login` y `/sign-up`
**Archivo:** `apps/web/app/sitemap.ts`
**Problema:** Estas paginas no aportan valor SEO.
**Fix:** Excluir del sitemap o poner `priority: 0`.

### Menores (P2)

#### 6. `strategy="beforeInteractive"` en JSON-LD innecesario
**Archivo:** `apps/web/app/page.tsx`
**Problema:** Bloquea el parsing del HTML. JSON-LD no es critico para el renderizado.
**Fix:** Cambiar a `strategy="afterInteractive"`.

#### 7. `error.tsx` sin funcion `reset`
**Archivo:** `apps/web/app/error.tsx`
**Problema:** El usuario no puede reintentar la accion sin recargar.

#### 8. `loading.tsx` sin texto alternativo
**Archivo:** `apps/web/app/loading.tsx`
**Problema:** El spinner animado no tiene `role="status"` ni texto accesible.

#### 9. Falta `viewport` metadata explicita
**Archivo:** `apps/web/app/layout.tsx`
**Problema:** Next.js 14 infiere el viewport por defecto, pero es mejor ser explicito.

---

## Acciones recomendadas

| Prioridad | Accion | Archivos |
|-----------|--------|----------|
| P0 | Agregar metadata con `robots: { index: false }` a paginas privadas | `dashboard/page.tsx`, `documents/page.tsx`, `emit/page.tsx`, `settings/page.tsx`, `documents/[id]/page.tsx` |
| P0 | Corregir `robots.ts` allow de `/home` a `/` | `robots.ts` |
| P0 | Asociar todos los `<label>` con `htmlFor` e `id` en inputs | `settings/page.tsx`, `emit-form.tsx` |
| P1 | Remover o marcar `aggregateRating` ficticio en JSON-LD | `page.tsx` |
| P1 | Limpiar sitemap de rutas no-SEO | `sitemap.ts` |
| P2 | Cambiar `beforeInteractive` a `afterInteractive` en JSON-LD | `page.tsx` |
| P2 | Agregar `role="status"` y texto a `loading.tsx` | `loading.tsx` |
